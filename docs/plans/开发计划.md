## 全局约定

### 术语口径

| 设计总纲用词 | 代码中用词 |
|-------------|-----------|
| Agent | Character |
| registration | characterMetaStore + characterContentStore |
| Soul | `soul.md` |
| Memory | `memory.md` |
| User | `user.md` |
| Sub-agent | Sub-agent（运行时概念，与 Character 不同） |

### 技术栈（已定）

- 后端: TypeScript + Hono + SQLite
- 前端: Vue 3 + TypeScript
- 交互: WebSocket (Socket.IO)
- 测试: Vitest

```typescript

  name: string

  avatar?: string

function checkStrategy(toolName: string, strategy: 'Plan' | 'Ask' | 'Bypass'): 'allow' | 'ask' | 'deny' {
  const dangerous = DANGEROUS_TOOLS.includes(toolName)

      solution: trajectory.finalAnswer,
      keySteps: trajectory.toolCalls,
      pitfalls: trajectory.errors,
    },
    tags: extractKeywords(trajectory),
    keyActions: trajectory.criticalActions,
  }
}
```

### 5.3 Skill 自动生成

```typescript
function generateSkill(insight: Insight, characterId: string): string {
  // 生成 SKILL.md
  const skillPath = `skills/generated/${characterId}/${slugify(insight.title)}/`
  mkdirSync(skillPath, { recursive: true })
  writeFileSync(skillPath + 'SKILL.md', renderSkillTemplate(insight), 'utf-8')
  // 注册到角色
  characterMetaStore.addSkill(characterId, skillPath)
  return skillPath
}
```

### 5.4 归属标记

技能文件路径绑定到角色：`skills/generated/{characterId}/{skillName}/`，目录归属即角色归属。

**验收：** 成功会话 ≥3 次 tool call 后自动生成 SKILL.md，角色可查看/使用自动生成的技能。
