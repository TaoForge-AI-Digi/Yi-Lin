## 一、总览

### 1.1 项目愿景

**「弈」(Yì)** 是一个融合了 OpenCode 执行流程、Hermes 人格进化与 Flowith 知识花园理念的 AI Agent 系统。其核心特征：

- **有灵魂**：每个 Agent 拥有独立的身份（Soul）、记忆与人格。
- **有策略**：用户可手动切换 Plan/Ask/Bypass 执行模式。
- **有分身**：主 Agent 可拉起同组的多个 Sub-agent 并行处理子任务。
- **有记忆**：知识以"典籍"形态存储，通过语义检索与图谱扩散召回。**（[Phase: 知识库]）**
- **会进化**：成功的执行轨迹可沉淀为 Skill 或新典籍。**（知识库入库标记延后）**

### 1.2 核心概念定义

| 概念 | 定义 |
|:---|:---|
| **Agent** | 拥有独立身份（Soul）、记忆、工具集、技能库和知识绑定的执行单元。 |
| **Main Agent** | 用户可直接对话的入口 Agent（`role = main` 或 `both`）。 |
| **Sub-agent** | 被其他 Agent 拉起的子执行单元（`role = sub` 或 `both`），拥有独立上下文。 |
| **Group** | Agent 的逻辑归属域。同组 Agent 可相互委托，跨组不可见。 |
| **Strategy** | 用户手动选择的执行策略：`Plan`（只读）、`Ask`（需确认）、`Bypass`（直接执行）。 |
| **Soul** | Agent 的人格定义文件（`SOUL.md`），固定存在于系统提示词顶层。 |
| **Skill** | 渐进式披露的标准化操作手册（`SKILL.md`），按需加载。 |
| **Sutra (典籍)** | **[Phase: 知识库]** 知识的最小原子单元，包含摘要、向量、原文位置与经络链接。 |
| **Meridian Link (经络链接)** | **[Phase: 知识库]** 典籍之间的语义关联（因果、参考、演进等）。 |

## 二、身份引擎

### 2.1 Agent 注册模型

每个 Agent 在系统中需注册以下信息：

```pseudocode
AgentRegistration:
    id: string                    // 唯一标识
    role: "main" | "sub" | "both" // Agent 角色能力
    group: string                 // 所属组名
    
    // 身份绑定
    identity:
        soul_path: string         // SOUL.md 文件路径
        memory_path: string       // MEMORY.md 文件路径
        user_memory_path: string  // USER.md 文件路径 (可选)
        
    // 能力绑定（权限 L1）
    bindings:
        skills: list<string>      // Skill 文件路径列表
        tools: list<ToolBinding>  // 工具绑定列表（含约束）
        knowledge: KnowledgeBinding? // [可选] 藏经阁连接配置
        
    // 策略默认值
    default_strategy: "Plan" | "Ask" | "Bypass"
```

### 2.2 工具绑定模型（含约束）

```pseudocode
ToolBinding:
    name: string                  // 工具名称
    constraints:                  // 参数级约束（替代独立 permission）
        allowed_paths: list<string>?   // 路径白名单（支持通配符）
        denied_paths: list<string>?    // 路径黑名单
        max_file_size: string?         // 最大文件大小
        allowed_commands: list<string>? // Bash 命令白名单
        denied_patterns: list<string>?  // Bash 命令黑名单
        readonly: boolean?              // 数据库只读
        max_rows: number?               // 查询最大行数
        require_confirm_even_in_bypass: boolean? // 强制确认
```

### 2.4 用户入口过滤

```pseudocode
FUNCTION list_agents_for_user():
    RETURN AGENT_REGISTRY.filter(agent => agent.role IN ["main", "both"])

FUNCTION validate_agent_for_user(agent_id):
    agent = AGENT_REGISTRY.get(agent_id)
    IF agent.role NOT IN ["main", "both"]:
        THROW "Agent '{agent_id}' 仅作为子代理，无法直接对话。"
    RETURN agent
```

### 2.5 身份加载流程

```pseudocode
FUNCTION load_identity(agent_id):
    reg = AGENT_REGISTRY.get(agent_id)
    
    RETURN {
        soul: read_file(reg.identity.soul_path),          // Stable层，永不压缩
        memory: read_file(reg.identity.memory_path),      // Stable层
        user_memory: read_file(reg.identity.user_memory_path) IF exists,
        skills_index: load_skills_summary(reg.bindings.skills), // Level 0
        tool_bindings: reg.bindings.tools,                // 含约束
        knowledge_config: reg.bindings.knowledge,         // 可选
        default_strategy: reg.default_strategy
    }
```

## 三、策略引擎

### 3.1 三种策略模式

| 模式 | 用户指令 | 允许操作 | 禁止操作 |
|:---|:---|:---|:---|
| **Plan** | `/plan` | `read`, `grep`, `glob`, `lsp`, `skill_load`(索引), `sutra_query` [Phase:知识库] | 所有写入 (`write/edit/delete`)、执行 (`bash`/`execute`)、`delegate_task` |
| **Ask** | `/ask` | 所有工具**均可调用**，但在调用写入/执行类工具前，**必须向用户请求确认** | 无（但确认环节可能被用户拒绝） |
| **Bypass** | `/bypass` | 所有工具无条件放行，无需确认 | 无（除非工具绑定了 `require_confirm_even_in_bypass`） |

### 3.2 策略状态存储

```pseudocode
SessionState:
    current_strategy: "Plan" | "Ask" | "Bypass"
    strategy_modified_by: "user" | "system"   // 标记由谁修改
    // 系统永不自切换策略，仅当用户输入命令时变更
```

### 3.3 用户切换命令

```pseudocode
FUNCTION handle_user_command(command, session):
    IF command == "/plan":
        session.state.current_strategy = "Plan"
        OUTPUT "[系统] 已切换到 Plan 模式（只读分析）"
    
    ELSE IF command == "/ask":
        session.state.current_strategy = "Ask"
        OUTPUT "[系统] 已切换到 Ask 模式（执行前需确认）"
    
    ELSE IF command == "/bypass":
        session.state.current_strategy = "Bypass"
        OUTPUT "[系统] 已切换到 Bypass 模式（直接执行）"
    
    ELSE IF command == "/status":
        OUTPUT "[系统] 当前策略: {session.state.current_strategy}"
```

### 3.4 危险工具分类（内置常量）

```pseudocode
// 全局常量，用于策略拦截
DANGEROUS_TOOLS = [
    "write", "edit", "delete", "patch",   // 文件写入类
    "bash", "execute", "mcp_exec",        // 命令执行类
    "delegate_task"                       // 子代理委托
]
```

### 3.5 策略继承（Sub-agent）

```pseudocode
FUNCTION determine_sub_strategy(parent_session, target_agent, user_specified):
    IF user_specified IS NOT NULL:
        RETURN user_specified
    // 默认继承主策略
    RETURN parent_session.state.current_strategy
```

## 四、调度引擎

### 4.1 双层循环总览

```
外层循环 (Session Loop)
├── 加载身份 (Soul + Memory + Skills索引 + 工具绑定)
├── [可选] 加载藏经阁知识
├── 组装 System Prompt
├── ┌──────────────────────────────────────────┐
│   │  内层循环 (Processor Loop)              │
│   │  ├── LLM 推理                           │
│   │  ├── 检测是否有 Tool Call               │
│   │  │   ├── 是 → 两阶段校验 → 执行 → 追加结果│
│   │  │   ├── 是 & tool == delegate_task     │
│   │  │   │   → 返回 Sub-agent 信号给外层    │
│   │  │   └── 否 → 返回最终答案              │
│   │  └── Doom Loop 检测                     │
│   └──────────────────────────────────────────┘
├── 处理 Sub-agent 信号 (拉起 → 回收 → 注入)
├── 检测是否需要上下文压缩 (OpenCode Compaction)
├── 检测是否触发顿悟 (Epiphany)
└── 循环继续 / 结束
```

### 4.2 外层循环

```pseudocode
FUNCTION session_loop(agent_id, user_input):
    // 1. 验证 Agent 可作主
    agent = validate_agent_for_user(agent_id)
    
    // 2. 加载身份
    identity = load_identity(agent_id)
    
    // 3. 初始化会话状态
    session = create_session(
        agent = agent,
        identity = identity,
        strategy = identity.default_strategy,
        history = []
    )
    
    // 4. [可选] 初始化藏经阁
    IF identity.knowledge_config IS NOT NULL:
        keeper = get_sutra_keeper(identity.knowledge_config)
    ELSE:
        keeper = NoOpSutraKeeper()  // 空实现 [Phase: 知识库]
    
    // 5. 主循环
    WHILE session.is_active:
        // 5.1 [可选] 取经：查询藏经阁
        knowledge_context = keeper.query(
            query = user_input,
            agent_id = agent_id,
            depth = 1
        )  // 如果 keeper 是 NoOp，返回空结果
        
        // 5.2 组装上下文
        prompt = assemble_prompt(
            soul = identity.soul,
            memory = identity.memory,
            skills_index = identity.skills_index,
            knowledge = knowledge_context,  // 可能为空
            strategy = session.state.current_strategy,
            history = session.history
        )
        
        // 5.3 调用内层循环
        result = inner_loop(session, prompt)
        
        // 5.4 处理内层返回结果
        IF result.type == "sub_agent_request":
            sub_result = spawn_and_run_sub_agent(
                task = result.task,
                target_agent = result.target_agent,
                parent_session = session,
                strategy_override = result.strategy_override
            )
            session.history.add_sub_agent_summary(sub_result)
            CONTINUE
        
        ELSE IF result.type == "final_answer":
            OUTPUT result.content
            session.is_active = False
        
        ELSE IF result.type == "need_compaction":
            session.history = compress_history(session.history)  // 参考OpenCode
            CONTINUE
    
    // 6. 后处理：顿悟检测
    IF session.completed_successfully:
        trigger_epiphany(session.trajectory, agent_id, keeper)
    
    RETURN session.final_answer
```

### 4.3 内层循环

```pseudocode
FUNCTION inner_loop(session, prompt):
    attempts = 0
    REPEAT:
        attempts += 1
        
        response = llm_generate(
            prompt = prompt,
            tools = session.identity.tool_bindings.map(_.name),
            strategy = session.state.current_strategy
        )
        
        IF response.has_tool_calls:
            FOR EACH tool_call IN response.tool_calls:
                // ★ 两阶段校验（见第七章）
                validate_tool_execution(tool_call, session)
                
                IF tool_call.name == "delegate_task":
                    RETURN {
                        type: "sub_agent_request",
                        task: tool_call.args.task,
                        target_agent: tool_call.args.target_agent_id,
                        strategy_override: tool_call.args.sub_strategy,
                        instances: tool_call.args.instances OR 1
                    }
                
                ELSE:
                    result = execute_tool(tool_call, session)
                    prompt.add_tool_result(result)
            
            // Doom Loop 检测 (参考 OpenCode)
            IF detect_doom_loop(prompt.tool_call_history):
                prompt.add_system_message(
                    "检测到连续失败，请尝试不同的方案或向用户澄清需求。"
                )
        
        ELSE:
            RETURN {
                type: "final_answer",
                content: response.content
            }
        
        IF attempts > MAX_INNER_LOOP_ATTEMPTS:
            THROW "内层循环超过最大尝试次数。"
```

### 4.4 Sub-agent 拉起与回收

```pseudocode
FUNCTION spawn_and_run_sub_agent(task, target_agent_id, parent_session, strategy_override):
    // 1. 获取目标 Agent
    target = AGENT_REGISTRY.get(target_agent_id)
    
    // 2. 组校验
    IF target.group != parent_session.agent.group:
        THROW "无法跨组委托。{target.id} 不属于 {parent_session.agent.group}。"
    
    // 3. 角色校验
    IF target.role NOT IN ["sub", "both"]:
        THROW "目标 Agent {target.id} 不具备子 Agent 角色能力。"
    
    // 4. 确定子策略
    sub_strategy = determine_sub_strategy(parent_session, target, strategy_override)
    
    // 5. 创建子会话（独立上下文）
    sub_session = create_sub_session(
        agent = target,
        parent_session_id = parent_session.id,
        strategy = sub_strategy,
        identity = load_identity(target.id),
        depth = parent_session.depth + 1
    )
    
    // 6. 递归深度限制
    IF sub_session.depth > MAX_SUB_AGENT_DEPTH:
        THROW "递归深度超过限制 ({MAX_SUB_AGENT_DEPTH})。"
    
    // 7. 子会话移除 delegate_task 防止孙代理
    IF sub_session.depth >= 1:
        sub_session.identity.tool_bindings.remove("delegate_task")
    
    // 8. [可选] 为子任务独立取经
    IF sub_session.identity.knowledge_config IS NOT NULL:
        sub_knowledge = get_sutra_keeper(sub_session.identity.knowledge_config).query(
            query = task,
            agent_id = target.id,
            depth = 1
        )
    ELSE:
        sub_knowledge = empty
    
    // 9. 组装子提示词
    sub_prompt = assemble_sub_prompt(
        system_identity = sub_session.identity.soul,
        memory = sub_session.identity.memory,
        knowledge = sub_knowledge,
        parent_context = parent_session.latest_context,
        task = task
    )
    
    // 10. 执行子任务（支持多实例）
    results = []
    instance_count = args.instances OR 1
    FOR i = 1 TO instance_count:
        instance_session = sub_session.clone()
        instance_result = inner_loop(instance_session, sub_prompt)
        results.append(instance_result)
        close_session(instance_session)
    
    // 11. 压缩回收结果 (参考 OpenCode summarizeSubResult)
    summarized = summarize_and_merge_results(results, max_tokens=2000)
    
    RETURN {
        summary: summarized.text,
        key_files: summarized.touched_files,
```

## 七、权限模型

### 7.1 设计原则

**「弈」不设独立的 `permission` 层。** 权限控制合并为**两阶段硬校验**：

| 阶段 | 归属引擎 | 检查内容 | 数据来源 |
|:---|:---|:---|:---|
| **L1: 身份绑定** | 身份引擎 | 工具是否在 `tool_bindings` 中 + 约束条件是否满足 | `AgentRegistration.bindings.tools` |
| **L2: 策略拦截** | 策略引擎 | 当前策略是否允许调用该类工具 | `SessionState.current_strategy` + `DANGEROUS_TOOLS` |

### 7.2 两阶段校验实现

```pseudocode
FUNCTION validate_tool_execution(tool_call, session):
    tool_name = tool_call.name
    args = tool_call.args
    
    // ========== L1: 身份引擎 - 工具绑定校验 ==========
    binding = session.identity.tool_bindings.find(t => t.name == tool_name)
    IF binding IS NULL:
        THROW "Agent 未绑定工具: {tool_name}"
    
    // L1 约束校验 (参数级)
    IF binding.constraints IS NOT NULL:
        // 路径约束
        IF "path" IN args:
            IF NOT match_any_pattern(args.path, binding.constraints.allowed_paths):
                THROW "路径 {args.path} 不在允许范围内"
            IF match_any_pattern(args.path, binding.constraints.denied_paths):
                THROW "路径 {args.path} 被禁止"
        
        // 文件大小约束
        IF "content" IN args AND binding.constraints.max_file_size:
            IF args.content.size > binding.constraints.max_file_size:
                THROW "文件大小超过限制"
        
        // Bash 命令约束
        IF tool_name == "bash" AND "command" IN args:
            cmd = args.command.split()[0]
            IF binding.constraints.allowed_commands:
                IF cmd NOT IN binding.constraints.allowed_commands:
                    THROW "命令 {cmd} 不在允许列表中"
            IF binding.constraints.denied_patterns:
                IF match_any_pattern(args.command, binding.constraints.denied_patterns):
                    THROW "命令包含禁止模式"
        
        // 数据库只读约束
        IF tool_name == "mcp:db_query" AND binding.constraints.readonly:
            IF args.query.trim().upper().starts_with("INSERT", "UPDATE", "DELETE", "DROP"):
                THROW "只读模式下禁止执行写操作"
        
        // 最大行数约束
        IF tool_name == "mcp:db_query" AND binding.constraints.max_rows:
            IF args.limit > binding.constraints.max_rows:
                THROW "查询行数超过限制"
    
    // ========== L2: 策略引擎 - 模式拦截 ==========
    strategy = session.state.current_strategy
    
    IF strategy == "Plan":
        IF tool_name IN DANGEROUS_TOOLS:
            THROW "Plan 模式禁止执行 {tool_name}。请切换至 Ask 或 Bypass 模式。"
    
    IF strategy == "Ask":
        IF tool_name IN DANGEROUS_TOOLS:
            // 检查是否强制确认（即使 bypass 也要确认）
            IF binding.constraints.require_confirm_even_in_bypass:
                user_decision = prompt_user("⚠️ 强制确认：执行 {tool_name}，参数：{args}？(y/n)")
            ELSE:
                user_decision = prompt_user("⚠️ 执行 {tool_name}，参数：{args}？(y/n)")
            
            IF user_decision != "y":
                THROW "用户已取消操作。"
    
    // Bypass: 放行（除非工具约束中强制确认）
    IF strategy == "Bypass":
        IF binding.constraints.require_confirm_even_in_bypass:
            user_decision = prompt_user("⚠️ 强制确认：执行 {tool_name}，参数：{args}？(y/n)")
            IF user_decision != "y":
                THROW "用户已取消操作。"
    
    RETURN pass
```

### 7.3 权限检查结果（与独立 permission 的对比）

| 场景 | 独立 permission 方案 | 「弈」两阶段方案 |
|:---|:---|:---|
| Agent 无此工具 | 需查 permission 表 | L1 直接拒绝 |
| Plan 模式调 write | 需查 permission 表 + 策略拦截 | L2 直接拒绝 |
| Ask 模式调 write | 需查 permission 表 + 询问用户 | L2 询问用户 |
| 写入 /etc/ 路径 | 需 permission 细粒度规则 | L1 约束校验拒绝 |
| Bash 执行 rm -rf | 需 permission 黑名单 | L1 约束校验拒绝 |

**结论**：独立 `permission` 层的所有功能，已被 L1 约束校验 + L2 策略拦截完全覆盖，且更简洁、更快速。

## 附录A：知识库接口契约 [Phase: 知识库]

以下为 `ISutraKeeper` 接口定义，供后续实现参考。核心引擎仅依赖此接口，不依赖具体实现。

```pseudocode
// ========================================
// 藏经阁接口契约 (ISutraKeeper)
// 具体实现将在 [Phase: 知识库] 阶段完成
// 当前阶段：核心引擎仅依赖 NoOpSutraKeeper 空实现
// ========================================

interface ISutraKeeper:
    // --- 取经：查询知识 ---
    // 输入：用户查询，Agent ID，检索深度
    // 输出：匹配的典籍摘要列表（渐进披露 Level 0）
    FUNCTION query(
        query: string,
        agent_id: string,
        depth: number DEFAULT 1,
        top_k: number DEFAULT 5
    ) -> SutraQueryResult
    
    // --- 取经：获取全文（Level 1）---
    // 当 Agent 决定引用某本典籍时调用
    FUNCTION get_full_sutra(
        sutra_id: string,
        agent_id: string
    ) -> SutraFullContent
    
    // --- 取经：定位原文来源（溯源）---
    // 输入：典籍 ID 或内容片段
    // 输出：原文位置信息
    FUNCTION locate_source(
        sutra_id: string,
        fragment: string OPTIONAL
    ) -> SourceLocation
    
    // --- 编纂：入库 ---
    // 输入：原始文档/洞察内容
    // 输出：入库的典籍 ID 列表
    FUNCTION compile(
        document: Document,
        agent_id: string,
        metadata: Metadata
    ) -> list<string>
    
    // --- 管理：归属性标记 ---
    FUNCTION mark_source_agent(
        sutra_ids: list<string>,
        agent_id: string
    ) -> void

// ========================================
// 数据结构
// ========================================

SutraQueryResult:
    summaries: list<SutraSummary>   // Level 0: 仅摘要
    total_count: number
    agent_id: string

SutraSummary:
    id: string
    title: string
    summary: string                 // 约 50 tokens
    tags: list<string>
    relevance_score: number

SutraFullContent:
    id: string
    title: string
    summary: string
    full_content: string            // Level 1: 完整原文
    source_location: SourceLocation // 溯源信息
    metadata: Metadata

SourceLocation:
    file_path: string               // 原始文件路径
    start_line: number OPTIONAL     // 起始行号
    end_line: number OPTIONAL       // 结束行号
    chapter: string OPTIONAL        // 章节名
    url: string OPTIONAL            // 网页链接
    snippet: string                 // 上下文片段（用于定位）
```

```pseudocode
// ========================================
// 早期开发：藏经阁空实现 (NoOpSutraKeeper)
// 所有查询返回空结果，不影响核心流程
// ========================================

class NoOpSutraKeeper implements ISutraKeeper:
    FUNCTION query(query, agent_id, depth=1, top_k=5):
        RETURN { summaries: [], total_count: 0, agent_id: agent_id }

        THROW "藏经阁功能未启用"
    
    FUNCTION locate_source(sutra_id, fragment=None):
        RETURN { file_path: "unknown", snippet: "" }
    
    FUNCTION compile(document, agent_id, metadata):
        RETURN []  // 无典籍入库
    
    FUNCTION mark_source_agent(sutra_ids, agent_id):
        // 无操作
```
